# BackPocket OS - API Documentation (Magic 2.0 Engine)

This document provides a comprehensive overview of the BackPocket OS API endpoints. The system is built on **FastAPI** and uses **SQLite (WAL mode)** for high concurrency.

---

## 1. Dashboard & Core System

### `GET /health`
- **Purpose:** Checks if the server is running and database is accessible.
- **Returns:** `{"status": "healthy", "version": "2.2", "pending": [...]}`

### `GET /dashboard`
- **Purpose:** Serves the main SPA (Single Page Application) dashboard (`static/index.html`).

---

## 2. Artificial Intelligence (Twins & Agents)

### `POST /api/twin-chat`
- **Purpose:** Unified chat endpoint for communicating with the Twin AI. The AI has local access to memory and pending approvals.
- **Payload:** `{"message": "string", "conversation_id": "string", "context": {}}`
- **Returns:** `{"response": "string", "conversation_id": "string", "title": "string"}`

### `POST /api/coach/analyze` (Phase 5)
- **Purpose:** Passes an email draft to the Communication Coach (GPT-4o via OpenRouter) for tone and style analysis.
- **Payload:** `{"ref_id": "string"}`
- **Returns:** 
  ```json
  {
    "status": "success",
    "analysis": {
      "confidence_score": 85,
      "feedback": ["Great tone", "Consider shortening sentence 2"],
      "power_version": "..."
    }
  }
  ```

### `GET /api/coach/yoodli` (Phase 5)
- **Purpose:** Returns the hook/URL for Yoodli Voice Practice integration.
- **Returns:** `{"status": "success", "yoodli_url": "https://app.yoodli.ai/practice", "message": "..."}`

## 3. Email Triage & Drafts (Human-in-the-Loop)

### `GET /api/pending`
- **Purpose:** Retrieves all pending emails (Tier 1 & 2) that require human approval before sending.
- **Returns:** `{"count": int, "items": [{"ref_id": "...", "sender": "...", "subject": "...", "tier": "..."}]}`

### `GET /api/draft/{ref_id}`
- **Purpose:** Get the AI-generated draft for a specific pending email.
- **Returns:** `{"draft": {"ref_id": "...", "draft_body": "..."}}`

### `POST /api/save-draft`
- **Purpose:** Manually save edits to an automatic draft.
- **Payload:** `{"ref_id": "...", "draft_body": "..."}`

### `POST /api/revise`
- **Purpose:** Send feedback to the AI Twin to rewrite a draft.
- **Payload:** `{"ref_id": "...", "comment": "Make it sound more professional"}`
- **Returns:** `{"status": "success", "new_draft": "..."}`

### `POST /api/approve`
- **Purpose:** Approve a draft and instruct Gmail to send it. **(The only way emails are sent)**.
- **Payload:** `{"ref_id": "..."}`

---

## 4. CRM & Data Flow

### `POST /api/add-client`
- **Purpose:** Manually add a client to the `Clients_Master` Google Sheet.
- **Payload:** `{"ref_id": "...", "first_name": "...", "last_name": "...", "email": "...", "mobile": "..."}`

### `POST /api/add-client-from-email`
- **Purpose:** Uses AI to extract Name, Phone, and Company from an email signature, and adds them to `Clients_Master`.
- **Payload:** `{"ref_id": "..."}`

---

## 5. Developer & Auto-Logging

### `POST /api/dev/auto-log`
- **Purpose:** System resilience hook. Appends a markdown-formatted message directly to `SESSION_LOG.md` or `docs/JOURNEY.md`.
- **Payload:** `{"content": "Completed phase 1...", "log_type": "session|journey"}`

### `Global Exception Handler`
- **Behavior:** Any `500 Internal Server Error` triggered by FastAPI is caught, the full traceback is parsed, and it is automatically appended to `docs/ERROR_LOG.md`.
